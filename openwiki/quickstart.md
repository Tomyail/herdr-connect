# Herdr Connect

Herdr Connect is an experimental, local-first companion for [Herdr](https://github.com/ogulcancelik/herdr) that enables mobile devices to discover and interact with AI agents running on the same local network. The current public milestone is **LAN Discovery Preview** — a deliberately limited scope that proves reachability without implementing pairing, authentication, or end-to-end encryption.

## What This Is

Herdr Connect consists of three main components:

1. **Go Daemon** — A background service that communicates with the Herdr CLI, maintains a local projection of agent state, and advertises itself on the LAN via Bonjour/mDNS
2. **iOS Mobile Client** — A React Native app that discovers the daemon, displays agent status, and provides limited interaction (view output, switch focus, send text input)
3. **Protocol Package** — TypeScript/Go cryptographic primitives for future secure pairing and end-to-end encryption (currently research-only, not integrated)

The daemon runs as a persistent service on macOS/Linux and communicates with Herdr through its CLI interface, parsing JSON output to track agents without embedding or linking Herdr source code.

## What This Is Not

Herdr Connect is **not**:

- A production-ready remote access product
- A cloud service or account system
- An authenticated or encrypted communication channel (in the current preview)
- A replacement for Herdr itself — it requires a separate Herdr installation

The current LAN demo exposes recent terminal output and text input over unencrypted HTTP. It must only run on trusted, controlled networks and should be stopped after testing.

## Current Scope

The **LAN Discovery Preview** milestone supports:

- ✅ Bonjour/mDNS daemon advertisement as `_herdr-connect._tcp`
- ✅ iOS discovery on physical devices (TestFlight beta available)
- ✅ Agent list display with status indicators
- ✅ View recent agent output (last 120 lines)
- ✅ Switch focus to an agent
- ✅ Send text input to an agent
- ✅ Localized UI (English and Chinese)
- ✅ Light/dark theme

Not yet implemented:

- ❌ Android app
- ❌ Pairing and device authentication
- ❌ End-to-end encryption
- ❌ Remote connections outside LAN
- ❌ Push notifications
